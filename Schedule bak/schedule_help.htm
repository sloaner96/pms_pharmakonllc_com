Scheduling Help


Selecting Meeting Dates
By default, all available business days will be selected as a meeting date. Click the days for which meetings will not be held. Days WITH meetings will be filled with a light blue color. Days WITHOUT meetings will be filled with a white color. If you have selected a range of months, you can navigate to a previous or following month by clicking the arrows on top of the main calendar. When you navigate months, clicking the arrow saves you work into the database.

Scheduling a Meeting
All Meeting Dates that were previously selected will appear on this page. Orange represents days available for meetings. White represents dates not available. Click the calendar of the month you would like to add meeting times to.

Scheduling Meeting Times
Click the day to schedule a meeting time. Moderators and Speakers will be provided based on their availability. Orange colored days are days when meetings can take place. Blue colored days represent days where times, moderators, and speakers are scheduled. The numbers below the date represent the number of meetings scheduled for that day. Gray colored days represent days when meetings will NOT take place.

Scheduling Meeting Times
All meeting times are displayed for day selected. To add a meeting, click "Add Meeting Time". Other buttons are explained below.
EDIT
  • Use the EDIT link to change the moderator and speaker. You can also CANCEL a meeting from this link. Meetings should only be cancelled if they were scheduled and did not occur. Cancelled meetings appear on reports for billing purposes.
DELETE
  • Use the DELETE link to delete a meeting completely from the system. Delete should be used if you need to change the time of a meeting or a mistake was made putting the meeting into the system.
Speaker ADD
  • For programs that have more than speaker, use this link to add additional speaker to the meeting.
ListenIns ADD
  • Add listen-ins to the meeting using this link.

Scheduling Moderator/Speaker Availability
Blue days are marked as available and white days are marked as unavailable. Click on a day to change its status. The color change will indicate the day's new status. Click save schedule and Exit when done. If you have selected a range of months, you can navigate to a previous or following month by clicking the arrows on top of the main calendar. When you navigate months, clicking the arrow saves you work into the database.

Scheduling Moderator/Speaker Time Availability
White days are marked as available and gray days are marked as unavailable. Days that a speaker or moderator is available only certain hours are indicated by the word "Partial". Click on any available day to see the speaker's or moderator's schedule in detail.

Scheduling Moderator/Speaker Time Availability Display
This page displays in detail a moderator's or speaker's schedule for a specific day. This table lists half hour increments and indicates if the person is available (A) or unavailable (NA). To edit these times, click the Edit Time button. The Close and Refresh button will close the time display pop-up and reload the Speaker Time Availability calendar. The Cancel button will close the time display pop-up with no reload.

Scheduling Moderator/Speaker Time Availability Edit
This table lists the same half hour increments displayed on the previous page, except this page has check boxes indicating available or unavailable. A checked box indicates available and an unchecked box indicates unavailable. To change a time's status, click or unclick the box. Click Save and Refresh when done or click Cancel to exit without saving changes. Times that have a meeting scheduled with this moderator or speaker will be uneditable. Before this time can be edited, the speaker or moderator must be removed from the meeting. Remove a speaker from a meeting by using menu item "Meeting Time Scheduler".

Viewing Meeting and Unavailable Schedules
This will report will allow you to view a Moderator's or Speaker's meeting/unavailability schedule for a given month. Only one month may be viewed at a time. Select the month and year you would like view. Then select the report type. Meeting Schedule will show all meeting codes and start times for the Moderator/Speaker selected. Times Unavailable will show all the times a Moderator/Speaker is Unavailable for the month selected. You must then select if the report is for a moderator or speaker. Based on your selection, a corresponding drop down list will appear. Only Speaker's/Moderator's who have participated in meetings will appear in the list. Choose the speaker/moderator and click "Generate Report". Be sure to change page layout to landscape when printing the report.

Production Schedule Reports
You can generate several reports from this screen. For each report when you must make a selection from the "Query By:" drop down box. This will determine what report will run. A list of each report is below. When you make a selection from the "Query By:" drop down, a second drop down list appears. Generally, you can choose to display "All" records matching your query, or you can select one project code, speaker, moderator, etc.

Project Meetings Report By Week
  • Displays meetings counts by project per week.
  • Select Query By: Project to run this report.
Moderator Status Report
  • Displays meeting counts per moderator by project. Totals are given for each moderator as well as each project.
  • Select Query By: Moderator to run this report.
Speaker Status Report
  • Displays meeting counts per speaker by project. Totals are given for each speaker as well as each project.
  • Select Query By: Speaker to run this report.
Meetings By Recruiter
  • Displays meeting for individual recruiter. PIW must be updated and complete for this report to pull data.
  • Select Query By: Recruiter to run this report.
Meetings By Teleconferencing Co.
  • Displays meeting for individual teleconferencing company. PIW must be updated and complete for this report to pull data.
  • Select Query By: Teleconferencing Co. to run this report.
Meetings By Client
  • Displays meeting for individual client. PIW must be updated and complete for this report to pull data.
  • Select Query By: Client to run this report.
